# Codex + Obsidian 学习流程改造设计

## 目标

将 Vault 从 Claudian 导向的“自动读写”提示词，调整为适合在 Codex 中协作、在 Obsidian 中沉淀与浏览的学习流程；同时让每日题目记录首先服务于复盘。

## 范围

只修改以下已有文件：

- `学习中枢.md`
- `00-配置/全局索引.md`
- `00-配置/学习日志.md`
- `00-配置/学员档案.md`
- `00-配置/进度看板.md`
- `03-学习笔记/Day01-哈希与双指针.md`

不修改 `.obsidian/`、原始素材、题目详解或其他每日笔记。

## 协作模型

Obsidian 是 Markdown 笔记、链接与回顾的阅读界面；Codex 是在仓库中读取上下文、引导解题、生成建议更新的协作教练。开始学习时，Codex 按“档案、日志、看板、当日笔记、相关知识页”的最小必要集合读取内容。结束时，它先给出一份拟写回清单；只有用户明确确认后，才更新学习日志、进度看板和当日笔记。

这消除了“每次必定自动写回”的不可靠承诺，同时仍保留完整学习闭环。

## 配置文件变更

- 学习中枢：身份与操作指令由 Claudian/Claude 改为 Codex；保留提示优先、框架归纳和 Wiki 链接规则；明确读取范围、确认式写回、Git/Obsidian Sync 备份提醒。
- 全局索引：更新核心导航和工具说明，使其反映 Codex + Obsidian 的分工。
- 学习日志：保留追加式历史，并在模板中加入“复盘要点”和“下次复习”字段。
- 学员档案：记录用户与 Codex 的协作偏好，即先提示、记录可追溯、写回须确认。
- 进度看板：保留统计和薄弱环节；将自动更新描述改为确认后更新，并增加可复习条目的入口。

## Day 1 题目记录结构

每道题使用相同的复盘顺序：

1. 题目目标
2. 重点
3. 时间/空间复杂度
4. 解题思路
5. 代码实现
6. 易错点与复盘
7. 掌握程度与下次复习
8. 示例

重点与复杂度在首屏，示例放在最后作为验证材料。已有题名、难度、当天模板和总结区域继续保留。
