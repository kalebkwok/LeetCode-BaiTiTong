## 二、补题原则

1. **先补轻题，建立手感**：Easy 和经典 Medium 优先，先让哈希、双指针、栈、二分这些基础模式顺起来。
2. **每题只补一篇高复盘价值题解**：重点、复杂度、核心模板、易错点放前面；例子放后面。
3. **每批结束后再更新专题状态**：新建题解后，把对应专题页里的状态从 `待建题解` 改成 `已有题解` 并补链接。
4. **图论和 DP 不要硬冲**：它们是后半段集中攻坚，不建议一开始连刷。

## 三、单题练习到题解的流程

> 核心想法：练题时不要一边写正式题解；先留下“解题现场”，AC 后再整理成可复盘的题解。

### 1. 练前：先判断题型，不急着写代码

打开题目后，先花 2~3 分钟写下：

| 要记录什么 | 你要问自己的问题 |
|---|---|
| 题目目标 | 最后到底要返回什么？索引、数量、布尔值、路径，还是结构本身？ |
| 输入规模 | `n` 大概多大？能不能 O(n²)？是否必须 O(n) / O(log n)？ |
| 题型猜测 | 这题像哈希、双指针、栈、二分、图、DP，还是设计题？ |
| 关键限制 | 有无重复、是否有序、能否原地修改、是否有负数、是否环形？ |

如果 3 分钟内完全没方向，就先写“我卡在哪里”，不要直接看完整题解。

### 2. 练中：记录转折点，而不是记录流水账

练题过程中只记三类信息：

| 事件 | 记录方式 |
|---|---|
| 想错了 | “我一开始以为 ___，但这个想法错在 ___。” |
| 卡住了 | “卡在 ___，原因是我没看出 ___。” |
| 想通了 | “关键转折是 ___，所以可以用 ___。” |

这三句话以后会直接变成题解里的“重点”和“易错点”。它们比复制一段官方思路更值钱。

### 3. 卡住时：分层求助

不要一上来就看完整题解。按这个顺序来：

1. **只看题型提示**：比如“这是滑动窗口 / 单调栈 / DFS”。
2. **只看核心不变量**：比如“窗口内最多替换 k 个字符”。
3. **只看伪代码**：看流程，不看完整代码。
4. **最后才看完整题解**：看完后必须合上，自己重写一遍。

如果需要我陪练，可以直接说：

```text
我现在练 424，只给我提示，不要直接给答案。
```

或者：

```text
我写完了，帮我只检查思路和边界，不要重写代码。
```

### 4. AC 后：立刻做 5 分钟复盘

AC 不等于掌握。每题做完后立刻补这 6 行：

```md
- 题型：
- 一句话思路：
- 关键不变量：
- 时间/空间复杂度：
- 我卡住的点：
- 下次看到类似题，先想：
```

这 6 行就是正式题解的原材料。

### 5. 从练习记录生成正式题解

正式题解按这个顺序写：

1. **题目目标**：这题到底要解决什么。
2. **重点**：最重要的 2~4 个观察。
3. **时间/空间复杂度**：先放前面，方便复盘。
4. **解题思路**：从暴力到优化，解释为什么这样做。
5. **代码实现**：放最终版本。
6. **易错点与复盘**：写自己真实卡过的地方。
7. **掌握程度 / 下次复习**：标记是否还需要二刷。
8. **示例**：最后放例子，用来验证流程。

## 十、每题完成定义

一题从 `待建` 变成 `完成`，需要同时满足：

1. 在 `02-Wiki/题目详解/` 新建题解文件；
2. 题解包含：题目目标、重点、时间/空间复杂度、解题思路、代码、易错点、例子；
3. 对应专题页的 `NeetCode 150 补充题目清单` 状态改为 `已有题解`，并链接到题解；
4. 如果这题暴露了薄弱点，写入 [[00-配置/进度看板|进度看板]] 的待复习清单。
