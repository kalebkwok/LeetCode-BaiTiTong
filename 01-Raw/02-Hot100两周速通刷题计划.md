# LeetCode Hot 100 两周速通刷题计划

> 适用对象：有 Python 基础的学习者
> 时间投入：14 天，每天 4-6 小时
> 目标：速通 Hot 100 并形成知识体系
> 方法论融合：labuladong（框架模板法）+ 代码随想录（系统专题法）+ 灵茶山艾府（题单驱动法）

---

## 一、总体策略

### 1.1 整体规划思路

14 天分为两个阶段：

- **第 1 周（Day 1 - Day 7）：打基础。** 以 Easy 和 Medium 为主，覆盖哈希、双指针、滑动窗口、数组、矩阵、链表、二叉树等基础数据结构专题。目标是建立模板意识，形成"看到题型就想到框架"的条件反射。
- **第 2 周（Day 8 - Day 14）：攻难点。** 以 Medium 和 Hard 为主，覆盖图论、栈、堆、回溯、二分查找、动态规划、多维 DP、技巧等进阶专题。目标是补齐短板，形成完整的算法知识体系。

### 1.2 每日时间分配建议（4-6 小时）

| 阶段 | 时间 | 内容 |
|------|------|------|
| 学习模板 | 30 分钟 | 先学习当天专题的核心模板/框架 |
| 看题理解 | 每题 5 分钟 | 读题、理解题意、分析数据范围 |
| 独立思考 | 每题 15-20 分钟 | 先想思路，不急着写代码 |
| 编码实现 | 每题 15-20 分钟 | 用 Python 写出可运行的代码 |
| 看题解优化 | 每题 10 分钟 | 对比题解，找更优解法 |
| 总结归纳 | 15 分钟 | 记录模板、易错点、心得体会 |
| 睡前复习 | 15 分钟 | 回顾今天所有题目的核心思路 |

### 1.3 刷题五步法

```
第一步：学模板 -- 每个专题开始前花 30 分钟学习核心模板
第二步：看题目 -- 仔细读题，标注关键条件和数据范围
第三步：想思路 -- 独立思考 15-20 分钟，写出伪代码或思路
第四步：写代码 -- 将思路转化为 Python 代码，注意边界条件
第五步：做总结 -- 对比题解，归纳模板，记录到笔记中
```

### 1.4 难度节奏

| 周次 | 难度分布 | 每日题量 | 重点 |
|------|---------|---------|------|
| 第 1 周 | Easy 为主 + 部分 Medium | 7-8 题/天 | 建立模板，熟悉套路 |
| 第 2 周 | Medium 为主 + 部分 Hard | 5-10 题/天 | 攻克难点，融会贯通 |

### Day 6：二叉树前 8 题（Easy + 简单 Medium）

**今日主题：** 二叉树基础——DFS 与 BFS 两大框架

**需要先掌握的模板/框架：**
- DFS（递归）框架：
```python
def dfs(node):
    if not node:
        return
    # 前序：处理当前节点
    dfs(node.left)
    # 中序：处理当前节点
    dfs(node.right)
    # 后序：处理当前节点
```
- BFS（层序）框架：
```python
from collections import deque
queue = deque([root])
while queue:
    for _ in range(len(queue)):
        node = queue.popleft()
        # 处理当前层节点
        if node.left: queue.append(node.left)
        if node.right: queue.append(node.right)
```

### Day 10：回溯（8 题）

**需要先掌握的模板/框架：**
- 回溯算法标准模板：
```python
def backtrack(path, choices):
    if 满足结束条件:
        result.append(path[:])  # 注意拷贝
        return
    for choice in choices:
        path.append(choice)     # 做选择

```
- 排列 vs 组合 vs 子集的区别：

  - 子集：组合的变体，每个组合都是一个结果

### Day 11：二分查找（6 题）

**今日主题：** 二分查找的多种变体与应用

**需要先掌握的模板/框架：**
- 标准二分查找模板：
```python
def binary_search(nums, target):
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = left + (right - left) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1
```
- 左边界查找：找第一个 >= target 的位置
- 右边界查找：找最后一个 <= target 的位置
- 关键：想清楚搜索区间是 [left, right] 还是 [left, right)

## 四、每日刷题模板

每日学习的标准流程如下，可根据实际时间灵活调整：

### 4.1 学习模板阶段（30 分钟）

- 阅读当天专题的核心模板和框架
- 理解模板的原理和适用场景
- 手写一遍模板代码，确保记住

### 4.2 逐题攻克阶段

对每道题执行以下流程：

| 步骤 | 时间 | 操作 |
|------|------|------|
| 读题理解 | 5 分钟 | 仔细阅读题目，标注输入输出格式、数据范围、特殊条件 |
| 独立思考 | 15-20 分钟 | 想出解法并写出伪代码。如果想不出，可以看提示但别看完整题解 |
| 编码实现 | 15-20 分钟 | 用 Python 实现，注意边界条件和特殊情况处理 |
| 测试调优 | 5 分钟 | 手动测试几个用例，提交后看运行时间和内存 |
| 看题解优化 | 10 分钟 | 阅读力扣题解，对比自己的解法，学习更优的思路 |

### 4.3 每日总结阶段（15 分钟）

- 记录每道题的核心思路（一句话版本）
- 标记做错的题和超时的题
- 整理新学到的模板或技巧

### 4.4 睡前复习阶段（15 分钟）

- 不看代码，回忆每道题的核心思路
- 用"费曼技巧"：假装给别人讲解每道题
- 回顾之前几天的题目（间隔复习）

## 七、应急方案

### 7.1 如果某天完不成怎么办

**优先级排序法：**

1. 先完成当天的 Easy 题，保证基础分
2. 再做核心 Medium 题（每个专题挑 2-3 道最经典的）
3. Hard 题可以跳过或留到周末补做
4. 实在来不及，把题解看一遍，记录思路，后续复习时重做

### 7.2 如果某道题卡住超过 30 分钟怎么办

**三步止损法：**

1. **前 20 分钟：** 独立思考，写出你能想到的所有思路，哪怕不完整
2. **20-25 分钟：** 看题目的"提示"（Hint），但别看完整题解
3. **25-30 分钟：** 如果还没有思路，直接看题解，理解后自己重新写一遍

**看题解的正确姿势：**
- 先理解思路，不要直接看代码
- 理解思路后，关掉题解，自己写代码
- 如果还是写不出来，再看代码，然后第二天重做这道题

**绝对不要做的事：**
- 死磕一道题超过 30 分钟（时间成本太高）
- 看完题解就过了（必须自己动手写一遍）
- 因为一道题卡住就放弃今天的计划（跳过它，继续后面的题）

### 7.3 保底刷题量

| 场景 | 保底题量 | 说明 |
|------|---------|------|
| 状态好的日子 | 全部完成 | 按原计划执行 |
| 状态一般的日子 | 完成 70% | 跳过最难的 1-2 道 Hard 题 |
| 状态差的日子 | 完成 50% | 只做 Easy + 核心 Medium |
| 时间不够的日子 | 完成 30% | 只做每个专题的必做题（见上表） |

**关键原则：每天至少做 3 道题。** 哪怕只做 3 道，也要保证认真做完、认真总结。质量比数量重要。

### 7.4 14 天后的持续学习计划

- **第 15-21 天：** 重做所有做错和超时的题目
- **第 22-28 天：** 限时模拟（每天 4 题，限时 2 小时）
- **面试前一周：** 每天随机抽 5 题复习，重点看高频面试题
