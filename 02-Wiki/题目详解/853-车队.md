# 853. 车队 (Medium)

> **专题归类：** [[02-Wiki/专题总结/06-栈与堆]] · [[02-Wiki/专题总结/11-贪心算法]]
> **LeetCode 链接：** https://leetcode.cn/problems/car-fleet/

---

## 题目目标

给定目标位置 `target`，以及每辆车的当前位置 `position[i]` 和速度 `speed[i]`，判断最终会形成多少个车队。

如果后面的车追上前面的车，它们会合并成一个车队，并以较慢的速度继续前进。

---

## 重点

- 按位置从靠近终点到远离终点排序。
- 每辆车到终点的时间是 `(target - position) / speed`。
- 后车如果到达时间小于等于前方车队时间，就会追上并合并。
- 只要出现一个更慢、更晚到达的时间，就形成一个新车队。

---

## 时间/空间复杂度

| 项目 | 复杂度 |
|---|---|
| 时间复杂度 | O(n log n) |
| 空间复杂度 | O(n) |

排序是主要成本。

---

## 解题思路

从终点往后看车。

越靠近终点的车，后面的车只能追它，不能被它追。所以先按 `position` 从大到小排序。

维护一个当前最慢车队的到达时间 `slowest_time`：

- 如果当前车到达时间 `time <= slowest_time`，说明它会追上前面的车队，车队数量不变。
- 如果 `time > slowest_time`，说明它追不上前面的车队，自己形成新车队。

这个过程有点像维护一个“到达时间单调递增”的栈，但实际上只需要保存最后一个车队时间。

---

## 代码实现

```python
from typing import List


class Solution:
    def carFleet(self, target: int, position: List[int], speed: List[int]) -> int:
        cars = sorted(zip(position, speed), reverse=True)

        fleets = 0
        slowest_time = 0

        for pos, spd in cars:
            time = (target - pos) / spd
            if time > slowest_time:
                fleets += 1
                slowest_time = time

        return fleets
```

---

## 易错点与复盘

- **排序方向要从靠近终点到远离终点。**
- **合并条件是 `time <= slowest_time`。** 后车更快或一样快，就会追上前面的车队。
- **不要模拟每一秒。** 这题本质是比较到达时间，不是物理运动模拟。

## 示例

```text
target = 12
position = [10, 8, 0, 5, 3]
speed    = [ 2, 4, 1, 1, 3]

按位置从大到小：
10 -> time = 1
8  -> time = 1
5  -> time = 7
3  -> time = 3
0  -> time = 12

10 和 8 合并为 1 个车队
5 自成 1 个车队
3 追上 5 的车队
0 自成 1 个车队

答案：3
```
