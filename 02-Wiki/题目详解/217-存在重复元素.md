# 217. 存在重复元素 (Easy)

> **专题归类：** [[02-Wiki/专题总结/01-哈希表]]
> **LeetCode 链接：** https://leetcode.cn/problems/contains-duplicate/

---

## 题目目标

给定整数数组 `nums`，判断其中是否存在任意一个元素至少出现两次。

---

## 重点

- 用哈希集合记录已经见过的数字。
- 扫描到一个已经在集合中的数字时，立刻返回 `True`。
- 这题的关键不是计数，而是“是否出现过”；集合比字典更贴合问题。

---

## 时间/空间复杂度

| 项目 | 复杂度 |
|---|---|
| 时间复杂度 | O(n) |
| 空间复杂度 | O(n) |

每个元素至多进行一次集合查询和一次插入。

---

## 解题思路

从左到右遍历数组，维护 `seen`：它表示“前面已经出现过的数字集合”。

当前数字 `num` 有两种情况：

1. `num in seen`：说明此前已经出现过，数组存在重复；
2. 不在：把它加入 `seen`，继续检查后面的数字。

这和两数之和的哈希表思想相同，都是利用哈希结构保存过去的信息；区别是这里不需要保存下标。

---

## 代码实现

```python
from typing import List


class Solution:
    def containsDuplicate(self, nums: List[int]) -> bool:
        seen = set()

        for num in nums:
            if num in seen:
                return True
            seen.add(num)

        return False
```

---

## 易错点与复盘

- **集合只存值。** 题目只问是否重复，不需要存下标或出现次数。
- **可以提前返回。** 找到第一个重复值就已经得到答案，不必遍历完整数组。
- `len(set(nums)) < len(nums)` 也可行，但循环写法更容易迁移到需要记录位置或次数的题。
