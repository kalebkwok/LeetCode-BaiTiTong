# 621. 任务调度器 (Medium)

> **专题归类：** [[02-Wiki/专题总结/06-栈与堆]] · [[02-Wiki/专题总结/11-贪心算法]]
> **LeetCode 链接：** https://leetcode.cn/problems/task-scheduler/

---

## 题目目标

给定任务列表 `tasks` 和冷却时间 `n`，同一种任务两次执行之间至少间隔 `n` 个单位时间。

求完成所有任务所需的最短时间。

---

## 重点

- 最高频任务决定整体框架。
- 先把出现最多的任务排开，中间留下冷却空位。
- 如果其他任务足够多，可以填满空位；否则需要 idle。
- 最终答案是公式结果和任务总数的较大值。

---

## 时间/空间复杂度

| 项目 | 复杂度 |
|---|---|
| 时间复杂度 | O(n) |
| 空间复杂度 | O(1) |

这里 `n` 表示任务数量；任务种类最多 26 个，所以额外空间是常数。

---

## 解题思路

假设出现最多的任务是 `A`，出现了 `max_freq` 次。

为了满足冷却时间，需要把这些 `A` 分成 `max_freq - 1` 个完整间隔：

```text
A _ _ A _ _ A
```

如果冷却时间是 `n`，每个间隔长度是 `n + 1`，最后一组只放最高频任务本身。

基础长度：

```python
(max_freq - 1) * (n + 1) + max_count
```

其中 `max_count` 是有多少个任务都达到了最高频。例如 `A` 和 `B` 都出现 3 次，最后一列要放 `A B` 两个任务。

最后还要和 `len(tasks)` 取最大值。因为如果其他任务很多，可以把 idle 全部填满，答案至少是任务总数。

---

## 代码实现

```python
from typing import List
from collections import Counter


class Solution:
    def leastInterval(self, tasks: List[str], n: int) -> int:
        count = Counter(tasks)

        max_freq = max(count.values())
        max_count = sum(1 for freq in count.values() if freq == max_freq)

        frame = (max_freq - 1) * (n + 1) + max_count
        return max(len(tasks), frame)
```

---

## 易错点与复盘

- **不是简单排序模拟。** 最短时间由最高频任务的冷却框架决定。
- **要考虑多个最高频任务。** `AAABBB` 且 `n=2` 时，最后一组不只是一个任务。
- **答案要和任务总数取最大值。** 其他任务足够多时，不需要 idle。
- **`n=0` 时直接返回任务总数。** 公式也能覆盖这种情况。

## 下次复习

重点复习公式来源：为什么是 `(max_freq - 1) * (n + 1) + max_count`。

---

## 示例

```text
tasks = ["A","A","A","B","B","B"], n = 2

A 出现 3 次，B 出现 3 次
max_freq = 3
max_count = 2

frame = (3 - 1) * (2 + 1) + 2 = 8

一种安排：
A B idle A B idle A B

答案：8
```
