# 743. 网络延迟时间 (Medium)

> **专题归类：** [[02-Wiki/专题总结/07-图论]]
> **LeetCode 链接：** https://leetcode.cn/problems/network-delay-time/

## 题目目标

从节点 `k` 发信号，求所有节点收到信号的最短时间；有节点不可达则返回 -1。

## 重点

- 正权最短路使用 Dijkstra。
- 堆中取出的最小距离若已访问，可以跳过。

## 时间/空间复杂度

| 项目 | 复杂度 |
|---|---|
| 时间复杂度 | O(E log V) |
| 空间复杂度 | O(V+E) |

## 代码实现

```python
import heapq
from collections import defaultdict
from typing import List


class Solution:
    def networkDelayTime(self, times: List[List[int]], n: int, k: int) -> int:
        graph = defaultdict(list)
        for a, b, time in times:
            graph[a].append((b, time))
        heap, seen, answer = [(0, k)], set(), 0
        while heap:
            time, node = heapq.heappop(heap)
            if node in seen:
                continue
            seen.add(node)
            answer = time
            for neighbor, weight in graph[node]:
                if neighbor not in seen:
                    heapq.heappush(heap, (time + weight, neighbor))
        return answer if len(seen) == n else -1
```

## 易错点与复盘

- `answer` 是最后一个被确认最短路的节点距离。
- 不可达节点会使 `seen` 数量小于 `n`。

## 示例

```text
[[2,1,1],[2,3,1],[3,4,1]] 从 2 发出，全部收到需 2。
```
