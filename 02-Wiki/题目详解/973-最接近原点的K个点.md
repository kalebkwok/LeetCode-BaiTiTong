# 973. 最接近原点的 K 个点 (Medium)

> **专题归类：** [[02-Wiki/专题总结/06-栈与堆]]
> **LeetCode 链接：** https://leetcode.cn/problems/k-closest-points-to-origin/

---

## 题目目标

给定平面上的点 `points`，返回距离原点 `(0, 0)` 最近的 `k` 个点。

距离用欧几里得距离，但比较大小时不需要开平方。

---

## 重点

- 比较距离时用 `x*x + y*y`，不用 `sqrt`。
- 要找最近的 `k` 个点，可以维护大小为 `k` 的最大堆。
- Python 只有小根堆，所以存负距离，让堆顶代表当前最远点。

---

## 时间/空间复杂度

| 方法 | 时间复杂度 | 空间复杂度 |
|---|---|---|
| 堆 | O(n log k) | O(k) |
| 排序 | O(n log n) | O(n) |

---

## 解题思路

如果直接排序所有点，可以做，但会多排很多不需要的点。

更适合 Top K 的做法是维护一个大小为 `k` 的堆：

1. 遍历每个点。
2. 计算平方距离。
3. 把 `(-dist, x, y)` 放入堆。
4. 如果堆大小超过 `k`，弹出堆顶。

因为堆里存的是负距离，距离越远，负数越小，越容易被弹出。

---

## 代码实现

```python
from typing import List
import heapq


class Solution:
    def kClosest(self, points: List[List[int]], k: int) -> List[List[int]]:
        heap = []

        for x, y in points:
            dist = x * x + y * y
            heapq.heappush(heap, (-dist, x, y))

            if len(heap) > k:
                heapq.heappop(heap)

        return [[x, y] for _, x, y in heap]
```

---

## 易错点与复盘

- **不用开平方。** 距离大小比较中，平方距离和真实距离的顺序一样。
- **负距离模拟最大堆。** 堆顶是当前保留点中最远的点，超过 k 时弹掉它。
- **返回顺序任意。** 题目不要求按距离排序。
