# 167. 两数之和 II - 输入有序数组 (Medium)

> **专题归类：** [[02-Wiki/专题总结/02-双指针与滑动窗口]]
> **LeetCode 链接：** https://leetcode.cn/problems/two-sum-ii-input-array-is-sorted/

---

## 题目目标

给定一个按非递减顺序排列、下标从 1 开始的整数数组 `numbers`，找出两个不同元素，使它们之和等于 `target`，返回它们的 1-based 下标。

---

## 重点

- 数组有序是双指针对撞的前提。
- 和太小就移动左指针，和太大就移动右指针。
- 返回值是 1-based 下标，最后要加 1。

---

## 时间/空间复杂度

| 项目 | 复杂度 |
|---|---|
| 时间复杂度 | O(n) |
| 空间复杂度 | O(1) |

每轮至少移动一个指针，两个指针最多各走完整个数组一次。

---

## 解题思路

初始时 `left = 0`、`right = len(numbers) - 1`。

因为数组有序：

- 若 `numbers[left] + numbers[right] < target`，左端再小也无用，必须增大和，所以 `left += 1`；
- 若和大于 `target`，右端再大也无用，必须减小和，所以 `right -= 1`；
- 相等时，当前这对下标就是答案。

这道题和 1. 两数之和的差别是：1 题没有排序，靠哈希表找过去；167 题利用排序，靠指针排除不可能的范围。

---

## 代码实现

```python
from typing import List


class Solution:
    def twoSum(self, numbers: List[int], target: int) -> List[int]:
        left, right = 0, len(numbers) - 1

        while left < right:
            total = numbers[left] + numbers[right]

            if total == target:
                return [left + 1, right + 1]
            if total < target:
                left += 1
            else:
                right -= 1

        return []
```

---

## 易错点与复盘

- **移动方向依赖有序性。** 无序数组不能依据和的大小安全移动指针。
- **返回的是 1-based 下标。** 内部仍用 Python 的 0-based 下标，返回时统一加 1。
- `left < right` 保证不会重复使用同一个元素。
