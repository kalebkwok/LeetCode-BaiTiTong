# 703. 数据流中的第 K 大元素 (Easy)

> **专题归类：** [[02-Wiki/专题总结/06-栈与堆]]
> **LeetCode 链接：** https://leetcode.cn/problems/kth-largest-element-in-a-stream/

---

## 题目目标

设计一个类，持续接收新的数字，并在每次加入新数字后返回当前数据流中的第 `k` 大元素。

---

## 重点

- 第 `k` 大不需要保存所有数字的完整排序。
- 用一个大小为 `k` 的小根堆维护当前最大的 `k` 个数字。
- 堆顶就是这 `k` 个数字中最小的，也就是整体第 `k` 大。

---

## 时间/空间复杂度

| 操作 | 时间复杂度 | 空间复杂度 |
|---|---|---|
| 初始化 | O(n log n) | O(k) |
| `add` | O(log k) | O(k) |

---

## 解题思路

维护一个小根堆 `heap`，里面只保留当前最大的 `k` 个元素。

每次加入新元素：

1. 把新元素入堆。
2. 如果堆大小超过 `k`，弹出堆顶最小值。
3. 此时堆里剩下的是最大的 `k` 个元素。
4. 堆顶就是第 `k` 大。

这就是 215 题「数组中的第 K 个最大元素」的在线版本。

---

## 代码实现

```python
from typing import List
import heapq


class KthLargest:

    def __init__(self, k: int, nums: List[int]):
        self.k = k
        self.heap = nums[:]
        heapq.heapify(self.heap)

        while len(self.heap) > k:
            heapq.heappop(self.heap)

    def add(self, val: int) -> int:
        heapq.heappush(self.heap, val)

        if len(self.heap) > self.k:
            heapq.heappop(self.heap)

        return self.heap[0]
```

---

## 易错点与复盘

- **用小根堆，不是大根堆。** 因为要保留最大的 `k` 个，方便把这 `k` 个里最小的踢掉。
- **堆大小必须维持为 k。** 如果超过 k 不弹出，堆顶就不再代表第 k 大。
- **初始化时也要裁剪。** `nums` 初始长度可能大于 k。

## 示例

```text
k = 3
nums = [4, 5, 8, 2]

保留最大的 3 个：[4, 5, 8]
堆顶 4 是第 3 大

add(3) -> 最大三个仍是 [4,5,8]，返回 4
add(5) -> 最大三个是 [5,5,8]，返回 5
add(10) -> 最大三个是 [5,8,10]，返回 5
```
