# 355. 设计推特 (Medium)

> **专题归类：** [[02-Wiki/专题总结/06-栈与堆]] · [[02-Wiki/专题总结/01-哈希表]]
> **LeetCode 链接：** https://leetcode.cn/problems/design-twitter/

---

## 题目目标

设计一个简化版 Twitter，支持：

- `postTweet(userId, tweetId)`：用户发布推文；
- `getNewsFeed(userId)`：返回该用户可见的最近 10 条推文；
- `follow(followerId, followeeId)`：关注；
- `unfollow(followerId, followeeId)`：取消关注。

---

## 重点

- 每条推文要记录时间戳，方便按新旧排序。
- 用户能看到自己和关注者的推文。
- `getNewsFeed` 只需要最近 10 条，不需要合并所有历史推文。
- 可以用堆从多个用户的最近推文中取最新的 10 条。

---

## 时间/空间复杂度

| 操作 | 时间复杂度 | 空间复杂度 |
|---|---|---|
| `postTweet` | O(1) | O(1) |
| `follow` / `unfollow` | O(1) | O(1) |
| `getNewsFeed` | O(F log F) 近似 | O(F) |

`F` 是关注人数。代码中每个用户最多取最近 10 条候选推文，所以实际常数较小。

---

## 解题思路

需要两个核心结构：

```python
tweets[user] = [(time, tweetId), ...]
following[user] = {followee1, followee2, ...}
```

发布推文时，给每条推文一个递增时间戳。

拉取新闻流时：

1. 找到用户自己和所有关注者；
2. 每个人最多取最近 10 条推文作为候选；
3. 用堆按时间从新到旧取出前 10 条。

这样不用把所有历史推文都扫描一遍。

---

## 代码实现

```python
from collections import defaultdict
import heapq


class Twitter:

    def __init__(self):
        self.time = 0
        self.tweets = defaultdict(list)
        self.following = defaultdict(set)

    def postTweet(self, userId: int, tweetId: int) -> None:
        self.time += 1
        self.tweets[userId].append((self.time, tweetId))

    def getNewsFeed(self, userId: int) -> list[int]:
        heap = []
        users = set(self.following[userId])
        users.add(userId)

        for user in users:
            for time, tweet_id in self.tweets[user][-10:]:
                heapq.heappush(heap, (-time, tweet_id))

        feed = []
        while heap and len(feed) < 10:
            _, tweet_id = heapq.heappop(heap)
            feed.append(tweet_id)

        return feed

    def follow(self, followerId: int, followeeId: int) -> None:
        if followerId != followeeId:
            self.following[followerId].add(followeeId)

    def unfollow(self, followerId: int, followeeId: int) -> None:
        self.following[followerId].discard(followeeId)
```

---

## 易错点与复盘

- **用户自己也要出现在新闻流里。**
- **取消关注不存在的人不能报错。** 用 `discard` 比 `remove` 更稳。
- **不要允许自己关注自己。** 虽然加了也不一定错，但会让逻辑变脏。
- **只要最近 10 条。** 每个用户取最近 10 条候选即可，旧推文不可能进入全局前 10。
