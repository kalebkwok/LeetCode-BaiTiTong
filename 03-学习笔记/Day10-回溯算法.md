### 回溯标准模板
```python
def backtrack(path, choices):
    if 满足条件:
        res.append(path[:])
        return
    for choice in choices:
        path.append(choice)
        backtrack(path, new_choices)
        path.pop()
```

# 排列：visited 标记

### 去重技巧
```python
# 排序 + 同层跳过
nums.sort()
for i in range(len(nums)):
    if used[i] or (i > 0 and nums[i] == nums[i-1] and not used[i-1]):
        continue
```

## 二、做题记录

### 1. 全排列（Medium）
- **核心思路：** 排列关心顺序，每一层都从所有 `nums` 中选；用 `used[i]` 标记当前路径里已经用过的元素。
- **代码实现：** `len(path) == len(nums)` 时记录 `path[:]`；递归前 `used[i] = True`，递归后 `used[i] = False`。
- **复杂度：** O(n * n!) / O(n)
- **掌握程度：** ✅
- **感悟/易错点：** 必须 `res.append(path[:])` 或 `path.copy()`，否则结果里保存的是同一个 `path` 引用，最后会都变成空列表。

### 2. 子集（Medium）
- **核心思路：** 子集不关心顺序，用 `start` 控制只往后选，避免把 `[1,2]` 和 `[2,1]` 重复计算。
- **代码实现：** 每个递归节点都是一个合法子集，进入 `backtrack(start, path)` 后立刻 `res.append(path.copy())`。
- **复杂度：** O(n * 2^n) / O(n)
- **掌握程度：** ✅
- **感悟/易错点：** 递归时传 `i + 1`，不是 `start + 1`；`start` 是下一层选择列表的起点。

### 3. 电话号码的字母组合（Medium）
- **核心思路：** 用 `index` 表示正在处理 `digits` 的第几位；每层只能从当前数字映射的字母集合中选。
- **代码实现：** `index == len(digits)` 时加入 `"".join(path)`；空输入要特判返回 `[]`。
- **复杂度：** O(n * 4^n) / O(n)
- **掌握程度：** ✅
- **感悟/易错点：** 这里的 `index` 不是从数组里任选，而是逐位处理数字字符串。

### 4. 组合总和（Medium）
- **核心思路：** 组合不关心顺序，所以用 `start` 防止顺序重复；数字可重复使用，所以递归传 `i`。
- **代码实现：** `remaining == 0` 记录答案；排序后若 `candidates[i] > remaining`，后面更大，直接 `break`。
- **复杂度：** O(指数级) / O(target)
- **掌握程度：** ✅
- **感悟/易错点：** 可重复选传 `i`，不可重复选才传 `i + 1`。

### 5. 括号生成（Medium）
- **核心思路：** 递归状态记录 `left` 和 `right`，分别表示已经放了多少个左括号和右括号。
- **代码实现：** `left < n` 时可以放左括号；`right < left` 时可以放右括号；长度到 `2 * n` 时记录答案。
- **复杂度：** O(4^n / sqrt(n)) / O(n)
- **掌握程度：** ✅
- **感悟/易错点：** 口诀：左不超 n，右不超左。

### 6. 单词搜索（Medium）
- **核心思路：** 用 `dfs(i, j, k)` 表示在格子 `(i, j)` 尝试匹配 `word[k]`，匹配后向四个方向搜索 `k + 1`。
- **代码实现：** 做选择是把当前格子标记为已访问，可以用 `visited` 集合，也可以原地改成 `"#"`；递归后必须还原。
- **复杂度：** O(m * n * 4^L) / O(L)
- **掌握程度：** 🔄
- **感悟/易错点：** 访问标记只属于当前路径；不还原会让其他起点或其他路径错过这个格子。

### 7. 分割回文串（Medium）
- **核心思路：** `start` 是下一刀起点，`end` 是当前尝试切出的这一段的终点。
- **代码实现：** 当前子串是 `s[start:end+1]`；如果是回文，加入 `path` 后递归 `backtrack(end + 1)`。
- **复杂度：** O(n * 2^n) / O(n)
- **掌握程度：** 🔄
- **感悟/易错点：** `end` 是当前层 `for` 循环的临时变量，不需要传给下一层；下一层会重新枚举自己的 `end`。

### 8. N 皇后（Hard）
- **核心思路：** 一行一层递归，所以不用检查 `row`；只检查列和两条对角线。
- **代码实现：** 维护 `cols`、`diag1 = row - col`、`diag2 = row + col` 三个集合；递归前加入，递归后删除。
- **复杂度：** O(n!) / O(n)
- **掌握程度：** ✅
- **感悟/易错点：** 主对角线 `row - col` 相同，副对角线 `row + col` 相同。

## 四、今日总结

**学到的新模板/技巧：**
- 回溯三步：做选择 -> 递归 -> 撤销选择。
- 先判断递归状态：排列用 `used`，子集/组合用 `start`，逐位组合用 `index`，网格搜索用 `(i, j, k)`，N 皇后用 `row`。
- 记录答案时要拷贝当前路径：`path[:]` 或 `path.copy()`。
- 重复元素去重：先排序，再用 `i > start and nums[i] == nums[i - 1]` 跳过同层重复分支。
- 组合总和对比：39 题可重复选传 `i`；40 题不可重复选传 `i + 1`。
- 子集对比：78 题输入无重复；90 题输入可能重复，需要同层去重。

**遇到的困难：**
- 131 中 `start` / `end` 的含义，以及为什么切片写作 `s[start:end+1]`。
- 79 中访问标记为什么必须在递归后还原。
- N 皇后中行天然由递归层保证，不需要额外检查 `row`。
- 40 / 90 中 `i > start` 的含义需要特别复盘：它表示只跳过同一层的重复起点。

**遗留问题（需复习）：**
- 复习 131：`start` 是跨递归传递的状态，`end` 只是当前层枚举的切割终点。
- 复习 79：用原地标记和 `visited` 两种写法各手写一次。
- 复习 51：手推 `row - col` 和 `row + col` 两条对角线。
- 复习 40 / 90：为什么同层重复要跳过，但不同层重复可以保留。
